01 · Outcome

Durable sessions

Sessions, turns, messages, active context, and pending queue entries survive process restarts in $HOME/.compass/state.db.

Safe conversation loop

One active turn per session. Tool calls finish with matching results before steering is injected; queued input begins only after the active turn settles.

Provider-neutral core

The harness depends on captured Effect AI models and tokenization services. OpenRouter wiring stays in an adapter and no model name is hard-coded.

02 · Vocabulary and invariants

TermMeaning
Session
The entire durable conversation: configuration, active context, archived pre-compaction history, turns, and pending inputs.
Turn
Starts with a prompt user message and includes assistant/tool exchanges until the model stops. Steering messages join the active turn; queued messages start later turns.
Message
A persisted Effect AI Prompt.Message. Tool calls and tool results remain first-class typed parts inside assistant/tool messages and first-class streamed events.
Safe boundary
A point after the current model response and every emitted tool call has a persisted result. Steering and tool-requested compaction are applied only here.
Core invariants: one session actor and one model generation per session; append before publish; tool call/result pairs never split; FIFO queue order; compaction swaps active context atomically; every terminal path emits a settled or failed event.
Nominal typing rule. Almost every domain value crossing a module or persistence boundary is backed by a branded Schema: session, turn, message, tool-call and pending-input IDs; ordinals; token counts and limits; timestamps; provider/model keys; and compaction thresholds. Commands, events, phases, roles, and statuses are closed tagged unions. Plain strings remain only for genuinely free-form content such as user text and generated summaries.

Input rules

  • Prompt starts only while idle.
  • Queue is durable FIFO and starts after the active turn; while idle it starts immediately.
  • Steer requires an active turn and is injected at the next safe boundary.

Instruction options

An injected Instructions service builds the system prompt effectfully from session context. Provide constructors for plain text, ordered sections, empty instructions, and fully custom builders.

Event contract

Schema-defined tagged events: session-opened, input-accepted, phase-changed, turn-started, message-started, response-part, message-committed, queue-changed, compaction-started/completed, turn-completed, failed, and settled. There is no snapshot API.

Steering

A steer command sets a durable pending flag. The current Effect AI stream continues through its current response and tool batch, so every tool call receives a result. At the boundary, the steer user message is appended to the same turn and the next model generation uses the updated context. Existing assistant output is retained.

Queued messages

Queued messages never alter the active turn. Once the model stops with no further tool work or steering, the turn is committed, the oldest queue item becomes a new turn, and the automatic token check runs before that turn’s first model call.

06 · Tools and compaction

execute

stub

A typed Effect AI tool accepting a command string and returning the stable generic result “Not implemented yet.” The handler is isolated so execution can be replaced later.

compact

control tool

Returns an acknowledgement tool result and marks compaction pending. The session loop performs compaction only after the tool batch reaches a safe boundary.

One compaction path for both triggers

  1. 1Trigger automatically immediately after a user message crosses the configurable threshold, or from the compact tool at the next safe boundary.
  2. 2Locate the newest user message. Send only the active prefix before it to the injected compaction model, using a dedicated system instruction and no tools.
  3. 3Generate plain summary text through Effect AI. Wrap it as an ordinary user Prompt.Message; prior summaries receive no special treatment on later compactions.
  4. 4In one SQLite transaction, append the summary and replace the active-context projection with [summary, newest user]. Older rows and the compact-tool exchange remain archived for audit but leave model context.
  5. 5Resume the same turn with the main model. If summarization fails, retain the original active context and emit a typed failure—never leave a half-compacted session.

Token counting uses an injected Effect AI Tokenizer when available and a configurable deterministic estimator fallback. Threshold, reserve, summary output limit, and compaction model are options; none are provider constants.

Storage layer. Define a provider-independent SessionStore Context service, then implement it with @effect/sql-sqlite-node, schema-decoded queries, transactions, and SqliteMigrator. An in-memory layer supports deterministic tests.
Recovery. Ensure $HOME/.compass through Effect FileSystem, use SQLite WAL, reclaim pending inputs on open, and convert any persisted streaming message to interrupted while retaining coherent text/tool parts.

packages/contracts/src/harness.ts is the single home for all shared harness Schema classes, branded identifiers, commands, events, persisted data shapes, options, and tagged errors. It uses branded schemas by default for semantically distinct primitives, and the contracts index re-exports the module.

Server modules import those contracts and own only Effect services, Layers, orchestration, provider adapters, and persistence implementations. They do not redefine public types or errors.

AgentSession owns only orchestration. Storage, instructions, models, token counting, and compaction remain injected services.

Models exposes conversation and compaction model services as captured Effect AI Model layers. A later provider adapter plugs in without changing the loop.

OpenRouter composes @effect/ai-openrouter, FetchHttpClient, redacted configuration, and the caller-selected model key.

index.ts exports the supported harness API and keeps internal machinery private.

    Create execute/compact Tool definitions, Toolkit handlers, token counter, compaction prompt, and OpenRouter adapter.

  4. 04

    Session actor

    Implement scoped command ingestion, keyed single-flight lifetime, streamed response/event handling, tool loop, steering, and queue drain.

  5. 05

    Compaction transaction

    Unify automatic and tool triggers, exclude the newest user message, atomically replace context, and resume generation.

  6. 06

    Verification and polish

    Complete fake-model and SQLite suites, document exports/configuration, then run repository-required checks.
